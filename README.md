## Overview

**CryptSV** is a web-based utility designed as the essential companion to the **QuizSV** proctoring system. While QuizSV focused on the secure recording and encryption of student activity during remote assessments, CryptSV serves as the evaluator's interface for data recovery and forensic analysis.

The application allows instructors and administrators to upload the encrypted output generated by the QuizSV desktop client. Using specific cryptographic protocols, CryptSV decrypts the captured metadata, behavioral logs, and media summaries, providing a structured report to verify whether academic integrity was maintained throughout the evaluation process.

---

## Core Features

* **Secure Decryption Engine:** Implements the necessary algorithms to unlock and read the tamper-resistant files generated by the QuizSV client.
* **Data Analysis Dashboard:** Provides a clear visualization of the captured session data, allowing evaluators to review specific timestamps and flags.
* **Behavioral Verification:** Facilitates the identification of anomalies or patterns that may suggest unauthorized collaboration or academic misconduct.
* **Client-Side Processing:** Designed to handle sensitive data efficiently within the browser environment, ensuring that the decryption process remains localized to the evaluator's terminal.

---

## Technical Relationship with QuizSV

CryptSV represents the "evaluator-side" of the remote proctoring ecosystem developed during the COVID-19 pandemic. The two projects function in tandem:

1.  **QuizSV (Client):** Captures, algorithmically verifies, and encrypts the assessment session.
2.  **CryptSV (Forensics):** Decrypts, visualizes, and analyzes the resulting integrity report.

## Installation and Usage

As a static web application, CryptSV can be run locally using any standard web server or by simply opening the entry point in a modern browser.

### 1. Clone the Repository

```bash
git clone [https://github.com/jclot/CryptSV.git](https://github.com/jclot/CryptSV.git)
cd CryptSV
```

### 2. Local Execution

You can serve the project using a simple HTTP server (such as the `serve` package or Python's built-in server):

```bash
# Using Python
python3 -m http.server 8000
```
Then, navigate to `http://localhost:8000` in your web browser.

### 3. Usage

1.  Launch the CryptSV web interface.
2.  Upload the encrypted file received from the **QuizSV** client.
3.  Enter the required credentials or keys (if applicable) to initiate the decryption process.
4.  Review the generated integrity analysis report.
